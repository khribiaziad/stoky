Uploading a Pickup PDF & Creating Orders
The main daily workflow — upload your Caleo pickup list and Stoky creates the orders.
📤
Scenario: New pickup list arrived from Caleo
You just downloaded the pickup PDF from your Caleo dashboard. You need to register these orders in Stoky and assign products to each one.
Admin · Confirmer
1
Go to Orders → click "Upload Pickup PDF"
On the Orders page, click the blue "📤 Upload Pickup PDF" button in the top-right. A file picker opens — select your Caleo PDF file.

Workflow 02
02
Processing Returns
When packages come back — either via a return PDF or manually one by one.
📥
Scenario: Caleo sent you a return list
Some orders were not delivered. Caleo provided a return PDF listing the CMD IDs that came back. You need to process them and put the stock back.
Admin · Confirmer
1
Click "Upload Return PDF"
On the Orders page, click "📥 Upload Return PDF". Select the return PDF from Caleo. Stoky reads all the CMD IDs from it.
2
Review matched orders
Stoky matches each CMD ID in the PDF to an order in the database. Unrecognized IDs are listed in red so you can investigate.
Return modal
📥 Process Returns — 2 matched
CMD-8330688
Ahmed Benali · Casablanca · 350 MAD
3
Mark sell bag / broken status, then confirm
For each returned order, optionally check "Sell Bag Returned" (adds 1 MAD back to expenses) or "Product Broken" (product goes to broken stock, not back to sellable inventory).
If "Product Broken" is checked, the item is NOT returned to available stock — it's logged in Broken Stock separately.
4
Click "Confirm Returns"
Orders are marked as "Cancelled" (returned). Stock is restored to the product variants. Capital recalculates automatically.
🔄
2 returns processed
Orders marked Cancelled · Stock restored · Capital updated
Alternative: Manual Return (one order at a time)
Click "↩ Return Order" button → Search by CMD ID or customer name → Select the order → Mark sell bag / broken status → Click "Confirm Return". Same result, one order at a time. Useful for single returns without a PDF.

Editing an Order
Fix mistakes or update any detail on an existing order.
✏️
Scenario: Wrong product was assigned, customer changed city
After creating the order you noticed the wrong size was selected, or the customer called to update their address and city.
Admin only
1
Find the order → click the ✏ edit button
On the Orders page, find the order in the table. In the Actions column, click the pencil (✏) button. The edit modal opens pre-filled with all current order data.
2
Make your changes
Update any field: customer name, phone, city, address, total amount, products, or expenses. Change products by selecting a different variant from the dropdown.

Click the "📄 Upload City PDF" button and select your carrier's fee schedule PDF. A progress bar shows parsing status — this can take up to 60 seconds for large files.
